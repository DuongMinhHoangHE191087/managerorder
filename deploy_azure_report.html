Quy Trình Deploy Bằng Giao Diện VS Code

Dựa trên hình ảnh bạn vừa chụp, tài khoản **Azure for Students** của bạn đã liên kết thành công. Bạn hãy thực hiện deploy theo các bước trực quan bằng chuột sau:

  • Bước 1: Build Docker Image lên Azure Container Registry (ACR)
    Bạn không cần tự chạy lệnh CLI hay cài đặt Docker cục bộ:

    🖱️ Thao tác chuột:

    1. Trong danh sách file dự án bên trái VS Code, tìm tệp Dockerfile (nằm ở thư mục gốc).
    2. Nhấp chuột phải vào file Dockerfile và chọn "Build Image in Azure...".
    3. VS Code sẽ hiện thanh thông báo ở trên:

    • Chọn Subscription: Azure for Students.
    • Chọn hoặc tạo mới Azure Container Registry (ACR): Đặt tên bất kỳ (Ví dụ: acrmanagerorder).
    • Chọn tag: latest.
    4. Nhấn Enter để Azure tự động build trực tiếp trên cloud. Sau khi xong, Image sẽ xuất hiện trong kho lưu trữ của bạn.

  • Bước 2: Tạo Mới Azure Container App từ Extension
    Khởi tạo môi trường vận hành trực tiếp từ menu bạn đang chụp ảnh:

    🖱️ Thao tác chuột:

    1. Mở tab **Azure Resources** (như ảnh bạn chụp).
    2. Tìm dòng Container Apps, bấm vào biểu tượng dấu cộng (+) ở góc trên thanh công cụ hoặc click chuột phải chọn "Create Container App...".
    3. Điền các thông số theo hướng dẫn ở thanh trên cùng của VS Code:

    • Tên Container App: managerorder-app.
    • Chọn Registry: Chọn kho ACR bạn đã tạo ở Bước 1.
    • Chọn Image và Tag: Chọn managerorder:latest.
    • Cấu hình cổng (Port): Nhập 8080 (vì Dockerfile gốc expose cổng 8080).
    • Chế độ Ingress: Chọn External (để mở cổng HTTPS công khai truy cập website).
    4. Nhấn Enter để khởi tạo. Azure sẽ mất khoảng 1-2 phút để dựng server.

  • Bước 3: Nhập Cấu Hình Biến Môi Trường (Environment Variables)
    Không cần gõ lệnh dài dòng, thêm trực tiếp các khóa API:

    🖱️ Thao tác chuột:

    1. Trên cây thư mục Azure Extension, bấm mở rộng mục Container Apps.
    2. Tìm Container App managerorder-app vừa tạo, click chuột phải và chọn "Edit Container App Configuration..." (hoặc mở trực tiếp trên cổng Azure Portal).
    3. Thêm các biến môi trường thiết yếu để kích hoạt hệ thống Web & Telegram Bot:

    • NODE_ENV = production
    • PORT = 8080
    • SUPABASE_URL = <đường-dẫn-supabase-của-bạn>
    • SUPABASE_SERVICE_ROLE_KEY = <key-dịch-vụ-supabase>
    • TELEGRAM_BOT_TOKEN = <token-bot-telegram-của-bạn>
    • TELEGRAM_RUNTIME_MODE = polling
    4. Lưu cấu hình, Container App sẽ tự động khởi động lại (restart) cực kỳ mượt mà.

Xem Log Hoạt Động Trực Tiếp

Để kiểm tra xem Telegram Bot và Next.js đã chạy thành công chưa:

Click chuột phải vào Container App managerorder-app trong extension, chọn "Start Streaming Logs". Toàn bộ log server sẽ hiển thị trực tiếp tại cửa sổ Output của VS Code vô cùng tiện lợi!
